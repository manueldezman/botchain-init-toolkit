Skill Introduction

A unified BOT Chain Skill that bundles the full developer workflow — environment health checks, contract scaffolding, compilation bridges, and testnet deployment — enabling any AI coding agent to execute the complete on-chain lifecycle from a single natural-language prompt.

MIT-0 · Free to use, modify, and redistribute. No attribution required.

Capabilities

Nine bundled capabilities cover the path from "write a contract" to "verified on explorer," across every supported IDE.

Single-Command Bootstrap

Upgrades a developer's machine into a full BOT Chain workstation via one npx command — no manual file moves or config edits.

Progressive Skill Directory

Drops token-efficient SKILL.md files, references, and asset templates into global registries for Claude Code, Codex, Cursor, and Windsurf.

Environment Doctor

Runs hidden path checks (which forge / npx hardhat) to confirm compiler availability before any build starts.

Auto-Correction Loop

If Foundry or Hardhat is missing, the agent asks for approval, then installs and restores the pipeline automatically.

Compiler Bridge

Executes local compilation frameworks (forge build / forge create) through a safe terminal execution tool.

Live Testnet Deploy

Signs and transmits compiled bytecode straight to the public BOT Chain Testnet using free faucet gas.

Explorer Fetcher

Queries the block explorer API for confirmation status and prints a verified contract link inside the chat.

Identity Registry Templates

Ships native AI Agent Identity primitives: wallet handle, blueprint DID, and boundary policy field limits.

Model-Agnostic Bridge

Routes inference through a litellm bridge so any endpoint can be swapped in via a local .env layout.

Skill File Storage Path

◆
Claude CodeGlobal skills directory
~/.claude/skills/

Claude Code loads global and project-level skills from its skills directory.

◆
CodexGlobal skills directory
~/.codex/skills/

Codex loads global skills from its skills directory and picks them up on a new session.

◆
CursorGlobal skills directory
~/.cursor/skills/

Cursor reads skill definitions from its skills directory. Project-level skills override global ones with the same name.

◆
WindsurfRules output
~/.codeium/windsurf/memories/global_rules.md .windsurf/rules/botchain-skill-engine.md

Windsurf is configured through Rules output instead of a global skills directory. The project-level Rules file is written only when the installer runs inside a real project.

Verify Skill Installation in Terminal

▸
Claude CodeCheck loaded skills
# Check loaded skills in Claude Code Type /skills to see loaded skills

Installed skills appear in the system reminder at session start, and can be checked in any new conversation.

▸
CodexCheck loaded skills
# Verify in Codex session Type /skills to list available skills

Codex loads skills on session start. A successful install appears in the available skills list on a new session.

▸
CursorCheck loaded skills
# Verify in Cursor session Run `skills list` to view available skills

If the BOT Chain skill appears in Cursor's installed skills list, the installation was picked up correctly.

▸
WindsurfCheck Rules output
# Verify Windsurf Rules output ls ~/.codeium/windsurf/memories/global_rules.md ls .windsurf/rules/botchain-skill-engine.md

Windsurf uses Rules output instead of a skills route. If the files are present but the tool does not pick them up, use the manual fallback in Settings → Cascade → Rules.

3

How Agent Uses Skill

STEP A

Write Your Prompt

Describe your task naturally. The agent scans the global SKILL.md and detects which skill to invoke.

You: "Create a standard BOT Chain identity contract and deploy it."
STEP B

Agent Runs Pre-Flight

The MCP server checks compiler health and, if needed, asks approval to auto-install missing tooling.

Agent: Using botchain-init-toolkit — checking Foundry availability...
STEP C

Skill Executes & Verifies

The skill compiles, signs, deploys to testnet, then pings the explorer API for a verified address.

Agent: Deployed. Verified at explorer.botchain.org/0x91a...4f2

Usage Examples

Each prompt below expands into the agent's actual execution trace — the same reasoning steps it runs when a skill fires for real.

01 Query Balance Thought for 2s
# Query the USDC balance for address {addr} 1. Match intent → skill: botchain-balance 2. Load reference → rpc-endpoints.md (BOT Chain testnet RPC) 3. Call check_environment_health() → RPC reachable 4. Query USDC balance for {addr} on Pharos {network_display_en} 5. Format result to human-readable balance # → returns balance in chat, no gas spent
02 Send Native Transfer Thought for 3s
# Send 0.0001 {native_token} to {to_addr} 1. Match intent → skill: botchain-transfer 2. Pre-flight: check_environment_health() → wallet key loaded 3. Build + sign transaction locally via MCP execute_terminal_cmd 4. Broadcast to Pharos {network_display_en} 5. Poll for confirmation, return tx hash + status
03 Deploy Contract Thought for 6s
# Deploy a simple storage contract, store/retrieve uint256 1. Match intent → skill: botchain-deploy 2. Copy boilerplate from assets/contract-template.sol 3. check_environment_health() → forge missing 4. Prompt user: "Foundry is missing. Install it?" → approved 5. Run foundryup, restore pipeline 6. execute_terminal_cmd("forge create --rpc-url ...") 7. fetch_explorer_state(tx_hash) → verified address returned
04 Batch Airdrop Thought for 4s
# Airdrop 1 USDC each to 3 addresses 1. Match intent → skill: botchain-batch-transfer 2. Resolve token contract {token_addr}, decimals: 6 3. Estimate gas per recipient via gas-estimation reference 4. Loop: sign + broadcast transfer() per address 5. Aggregate tx hashes, return batch summary
05 Contract Read — Total Supply Thought for 2s
# Call totalSupply() on USDC contract 1. Match intent → skill: botchain-read-contract 2. Load ABI fragment for totalSupply() 3. eth_call (read-only, no signature required) 4. Format result in human-readable units
06 Gas Estimation Thought for 3s
# Estimate gas for transfer(address,uint256), 100 USDC 1. Match intent → skill: botchain-gas-estimate 2. Simulate call via eth_estimateGas 3. Cross-check against current base fee 4. Return estimated cost + failure-risk flag
07 Check Transaction Status Thought for 2s
# Query status of transaction {tx_hash} 1. Match intent → skill: botchain-tx-status 2. fetch_explorer_state(tx_hash) 3. Parse status, from, to, value, gas used 4. Present transaction details inline
